# System Design References

This directory keeps the LessonOps system-design reference pack in the
chezmoi-managed `dot_agents` tree so it is installed to
`~/.agents/system-design/` for agent work that touches LessonOps-style
product, architecture, UI, AI workflow, source-lineage, or
education-platform decisions.

Use these docs as domain references. They do not override active-session
instructions, the repo AGENTS precedence chain, or narrower instructions in a
working repository.

## Reference Map

- [Architecture](architecture.md): platform architecture, hosting, tenancy,
  data flow, AI validation, source library, assignments, analytics, security,
  observability, and CI/CD.
- [Coding Standards](coding-standards.md): app, API, database, frontend,
  testing, security, review, documentation, and AI-agent workflow standards.
- [UI Design](ui-design.md): Teacher/Admin Portal, Student Portal contract,
  design tokens, screen structure, review flows, permissions, and UI quality
  gates.
- [Knowledge Base](knowledge-base.md): Obsidian-style Knowledge Vault storage,
  note format, graph links, review workflow, and AI context bundles.
- [Student Practice Packages](student-practice-packages.md): published
  student package storage, Cloud Storage delivery, versioning, answer keys,
  and online/offline behavior.
- [Trusted Sources](trusted-sources.md): source trust, rights metadata,
  categorization, approval rules, and curriculum/source governance.
- [Business Strategy](business-strategy.md): LessonOps product positioning,
  customer model, domain strategy, roadmap, pricing, and market framing.
